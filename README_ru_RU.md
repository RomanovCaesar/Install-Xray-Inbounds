# Набор инструментов Caesar для установки и управления Xray в один клик (Install-Xray-Inbounds)

Это мощный, высокомодульный и максимально совместимый набор скриптов для установки и управления узлами Xray. Он поддерживает идеальное сосуществование нескольких основных протоколов (VLESS-Reality, VLESS-Encryption, Shadowsocks 2022 и т.д.) на одном сервере, а также предлагает удобное резервное копирование конфигурации, управление маршрутизацией (распределением трафика) и функцию обновления данных Geo.

## ✨ Основные возможности

  * **Умное сосуществование нескольких протоколов**: Использует базовый парсинг `jq` для интеллектуального добавления JSON-конфигураций. Устанавливайте несколько различных протоколов или многопортовых узлов по своему желанию — скрипт **абсолютно не перезапишет** конфигурации оригинальных узлов.
  * **Исключительная совместимость с системами**: Не только идеально поддерживает основные системы на базе Systemd, такие как Debian / Ubuntu, но и имеет **глубокую совместимость с Alpine Linux (OpenRC)**, что делает его одинаково удобным для чрезвычайно минималистичных и легких систем.
  * **Дружелюбность к NAT / DDNS**: Встроена функция независимой настройки адреса подключения. Независимо от того, используете ли вы машину с динамическими портами NAT или разрешение доменных имен через DDNS, вы можете в один клик сгенерировать правильную ссылку для обмена, забыв о необходимости ручного изменения адреса.
  * **Поддержка передовых протоколов**: Поддерживает новейшие протоколы VLESS Encryption (постквантовое чистое шифрование Post-Quantum) и VLESS-Reality (Vision), автоматически преобразуя нативные ключи в более совместимые случайные ключи.
  * **Управление "всё в одном"**: Предоставляет единое глобальное меню управления (`xray-manager`), инструмент настройки маршрутизации (`xray-routing`), а также инструмент резервного копирования и восстановления конфигурации (`xray-restore`).
  * **Безопасное и точное удаление**: Поддерживает точную идентификацию и удаление конфигураций определенных узлов по порту и протоколу, полностью исключая случайное повреждение других конфигураций.

## 🚀 Быстрый старт (Рекомендуется)

Если вы хотите получить доступ к полному набору функций управления, рекомендуется установить **Единый центр управления (Xray Manager)**.

Выполните следующую команду, чтобы скачать и вызвать глобальное меню управления:

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/xray_manager.sh -o xray_manager.sh && chmod +x xray_manager.sh && sudo ./xray_manager.sh
```

**💡 Полезный совет:**
После установки единого инструмента управления автоматически регистрируется глобальная команда. В дальнейшем вы можете в любой момент быстро вызвать главное меню, введя в терминале следующую команду:

```bash
xray-manager
```

В меню `xray-manager` вы можете запускать все нижеперечисленные независимые функции в один клик, без необходимости загружать скрипты по отдельности.

## 📦 Руководство по независимой установке каждого модуля

Если вы хотите использовать только определенную функцию этого проекта, вы также можете напрямую использовать следующие команды для независимой установки.

### 1\. Управление узлами VLESS-Reality (Vision)

Поддерживает автоматическую генерацию пар ключей X25519, по умолчанию использует управление потоком `xtls-rprx-vision`, работает стабильно и скрытно.

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/install_vless_reality.sh -o install_vless_reality.sh && chmod +x install_vless_reality.sh && sudo ./install_vless_reality.sh
```

### 2\. Управление узлами VLESS-Encryption (Post-Quantum)

Развертывает новейший протокол постквантового шифрования (ML-KEM-768), автоматически генерирует и оптимизирует случайные ключи, защищая будущую сетевую безопасность.

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/install_vless_encryption.sh -o install_vless_encryption.sh && chmod +x install_vless_encryption.sh && sudo ./install_vless_encryption.sh
```

### 3\. Управление узлами Shadowsocks 2022 и традиционными SS

Поддерживает сверхбыстрые протоколы шифрования нового поколения, такие как 2022-blake3-aes, а также имеет обратную совместимость с традиционным шифрованием aes-gcm, автоматически генерируя надежные случайные пароли.

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/install_ss2022.sh -o install_ss2022.sh && chmod +x install_ss2022.sh && sudo ./install_ss2022.sh
```

### 4\. Серверный инструмент маршрутизации (Xray Routing)

Мощная панель управления исходящим трафиком (маршрутизацией) на стороне сервера. Поддерживает импорт внешних ссылок на конфигурацию, поддерживает парсинг различных параметров в ссылках ss и vless (в настоящее время поддерживает только ss, ss2022, vless vision reality и vless encryption), визуальную настройку правил распределения IP/доменов от Inbounds к Outbounds.

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/xray_routing.sh -o xray_routing.sh && chmod +x xray_routing.sh && sudo ./xray_routing.sh
```

*После установки всегда доступен для вызова командой `xray-routing`.*

### 5\. Инструмент резервного копирования и восстановления (Xray Restore)

Случайно ошиблись в настройках? Хотите перенести конфигурацию? С помощью этого инструмента вы можете импортировать файл конфигурации по прямой ссылке URL или открыть консоль для ручной вставки `config.json`, инструмент имеет встроенную функцию тестирования безопасности для предотвращения ошибок запуска.

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/xray_restore.sh -o xray_restore.sh && chmod +x xray_restore.sh && sudo ./xray_restore.sh
```

*После установки всегда доступен для вызова командой `xray-restore`.*

### 6\. Инструмент полного удаления

Если вы столкнулись с неразрешимой серьезной проблемой или хотите полностью очистить сервер, вы можете использовать этот скрипт. Он предельно чисто удалит системные службы (Systemd/OpenRC), бинарные файлы, логи и остаточные конфигурации.

```bash
curl -L https://raw.githubusercontent.com/RomanovCaesar/Install-Xray-Inbounds/main/uninstall_xray.sh -o uninstall_xray.sh && chmod +x uninstall_xray.sh && sudo ./uninstall_xray.sh
```

## 🛠️ Часто задаваемые вопросы (FAQ)

**Q: Мой сервер — это NAT VPS, или входящий IP отличается от исходящего IP, из-за чего сгенерированный узел не работает. Что делать?**

**A:** Пожалуйста, выберите опцию **«Установить адрес подключения (NAT/DDNS)»** в любом меню установки (или в главном меню управления). Введите IP-адрес или доменное имя DDNS, которое вы фактически используете для внешнего подключения. После завершения настройки ссылка для обмена, автоматически сгенерированная скриптом, будет использовать указанный вами адрес, что идеально решает проблему прямого подключения в средах NAT.

**Q: Как посмотреть логи работы Xray?**

**A:** В меню каждого скрипта установки и управления есть опция **«Просмотр логов Xray»**. Выбрав ее, вы сможете просматривать логи работы в реальном времени. Нажмите `Ctrl + C`, чтобы остановить просмотр и вернуться в меню.

**Q: Как обновить файлы правил маршрутизации GeoIP и GeoSite?**

**A:** Если вы используете `xray-routing` (серверный инструмент маршрутизации), он включает функцию автоматической настройки запланированных задач в один клик, которая будет автоматически обновлять их каждый день на рассвете. Вы также можете выполнить мгновенное обновление вручную в главном инструменте управления (`xray-manager`).
